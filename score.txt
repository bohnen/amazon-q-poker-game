AWS PORKER - スコア表
====================

【基本ルール】
- 5枚のカードで役を作る
- より高い役ほど高得点
- 同じ役の場合は、より希少なスート・ランクの組み合わせが高得点

【通常の役】（低い順）

1. ハイカード (High Card) - 10点
   役なしの場合、最も高いランクのカードで判定
   
2. ワンペア (One Pair) - 50点
   同じランクのカード2枚
   
3. ツーペア (Two Pair) - 100点
   異なるランクのペア2組
   
4. スリーカード (Three of a Kind) - 200点
   同じランクのカード3枚
   
5. ストレート (Straight) - 400点
   連続する5つのランク（A-2-3-4-5 から 9-10-J-Q-K）
   
6. フラッシュ (Flush) - 基本点数 + スート別ボーナス
   同じスートのカード5枚
   - Greenフラッシュ: 2000点 (最も希少、24枚のみ)
   - Yellowフラッシュ: 1800点 (27枚のみ)
   - Orangeフラッシュ: 1600点 (30枚のみ)
   - Redフラッシュ: 1000点 (75枚)
   - Purpleフラッシュ: 800点 (82枚)
   - Blueフラッシュ: 600点 (114枚)
   - Grayフラッシュ: 500点 (最も多い、175枚)
   
7. フルハウス (Full House) - 1200点
   スリーカード + ワンペア
   
8. フォーカード (Four of a Kind) - 2500点
   同じランクのカード4枚
   
9. ストレートフラッシュ (Straight Flush) - 基本5000点 + スート別ボーナス
   同じスートで連続する5つのランク
   スート別ボーナス（フラッシュと同じ倍率）

【AWSスペシャル役】

10. クラウドトリオ (Cloud Trio) - 800点
    Compute(Green) + Storage(Blue) + Database(Blue) の組み合わせ
    （異なるランクの3枚 + 他2枚）
    
11. セキュリティスイート (Security Suite) - 1500点
    Security(Yellow)のフラッシュ + 特定の組み合わせ
    Yellow 3枚 + Red 1枚 + Purple 1枚
    
12. IoTエコシステム (IoT Ecosystem) - 1000点
    IoT(Red) 2枚 + Analytics(Orange) 2枚 + AI(Purple) 1枚
    
13. サーバーレスコンボ (Serverless Combo) - 1300点
    Lambda関連アイコン + API Gateway関連 + DynamoDB関連
    （特定のファイル名パターンを含む組み合わせ）
    
14. マルチクラウド (Multi-Cloud) - 2200点
    7つ全てのスートから最低1枚ずつ（5枚なので不可能だが、
    実際は5つの異なるスートでボーナス）
    
15. AWSアーキテクト (AWS Architect) - 3000点
    Management(Gray) + Security(Yellow) + Networking(Purple) + 
    Compute(Green) + Storage(Blue) の5枚
    
16. レジェンダリーフラッシュ (Legendary Flush) - 10000点
    最も希少なGreenスートでのストレートフラッシュ
    
17. AWSマスター (AWS Master) - 15000点
    A-K-Q-J-10 のロイヤルストレートフラッシュ
    （どのスートでも成立、スート別ボーナス適用）

【ボーナス計算】
- 希少スートボーナス: Green(x3), Yellow(x2.5), Orange(x2)
- 特定サービス組み合わせボーナス: 該当する場合 +200点
- 連続ランクボーナス: ストレート系で +100点

【判定優先順位】
1. 最高位の役を適用
2. 同じ役の場合、より希少なスート・ランクを優先
3. AWSスペシャル役は通常役より優先
4. 複数のスペシャル役が成立する場合、最高点を適用

【特記事項】
- カード総数527枚の不均等分布を考慮した配点
- 実際のAWSサービスの関連性を反映したスペシャル役
- ゲームバランスを考慮し、極端に高得点にならないよう調整
